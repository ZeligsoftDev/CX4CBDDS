Tutorial 2: Top-down Design

This tutorial assumes that you have completed Tutorial 1: Getting Started.

In this introductory tutorial you will learn how to design the application from the top-down and will:

  • Create a Block Diagram of the Application
  • Define the Components
  • Connect the Components
  • Define and Add the CORBA Interfaces
  • Define and Add the DDS Connectors

You can view a video of this tutorial here.  Depending on your screen size, you may want to close the Contents frame.

Create a Block Diagram of the Application

Block diagrams provide a high-level view of the application.  They can be used to design the overall structure of an application or component and then refined by adding details.

To create a block diagram of DiningPhilosophers, follow these steps:

  1. In the Project Explorer, right-click on the  DiningPhilosophers model.

  2. From the context menu, select Add CX > CCM Component.
    The tool opens the New Component wizard.

  3. Name the component DiningPhilosophers.

  4. Ensure Create Assembly and the other options are checked

  5. Rename the structure diagram to DiningPhilosophers_asm.

  6. Click Finish.

  7. Save your model.

Add Parts to the Block Diagram

You need to add blocks representing the four philosophers and their forks.

To add parts to the block diagram, follow these steps:

  1. From the Palette, drag-an-drop CCMPart onto the diagram and name it philosopher0.

  2. Repeat this three more times so that you have four components named philosopher0, philosopher1, philosopher2, and philosopher3.

    Note    : Pressing CTRL after clicking a Palette icon allows you to multi-drop model elements onto a diagram.

    Note    : You can use Edit > Duplicate or copy and paste to duplicate a model element.

     

  3. Add four more parts to the structure diagram: fork0, fork1, fork2, and fork3.

  4. Arrange the components so that they look somewhat akin to the following:



    Note    :  You can arrange and align elements using the tools under the Diagrams menu or on the toolbar.

  5. Save your model.

Define the Components

We will now model the two types of components (Fork and Philosopher) and add ports and attributes.

Add Component Fork

To add the Fork component, follow these steps:

  1. Right-click DiningPhilosophers and from the context menu select Add CX > CCM Component.
    The tool displays the New Component wizard.

  2. Ensure Create Assembly is not checked.

  3. Ensure that Create component in a new package and Add component to a new diagram are checked.

  4. In the Component Name field, type Fork.

  5. Name the diagram Fork_diagram.

  6. Click Finish.
    The tool adds the component to the model.

  7. Save your model.

Add Component Philosopher

To add the Philosopher component, follow these steps:

  1. Right-click DiningPhilosophers and from the context menu select Add CX > Component.
    The tool displays the New Component wizard.

  2. Ensure Create Assembly is not checked.

  3. Ensure that Create component in a new package and Add component to a new diagram are checked.

  4. In the Component Name field, type Philosopher.

  5. Name the diagram Philosopher_diagram.

  6. Click Finish.
    The tool adds the component to the model.

  7. Save your model.

Create Component Definitions

To create component definitions, follow these steps:

  1. In the DiningPhilosophers_asm diagram, select all the philosopher blocks.

  2. Right-click and from the context menu, select Filters > Typed Element Label Style > Element Name Only (optional).

    Note    : You can display labels in any of the following three ways:

    • Element and Type Names

    • Element Name Only

    • Type Name Only
       

  3. In the Properties tab below the diagram, select the DDS4CCM definition and click (Edit Definition).
    The tool opens the Element Selection Dialog.

  4. Select the Philosopher component.

  5. Click OK.

  6. Repeat for the fork components.

  7. Save your model.

Connect the Components

Components communicate via Interface Ports.

Add Interface Ports to forks

To add Interface Ports to the fork components, follow these steps:

  1. From the Palette, drag-and-drop an Interface Port onto a fork.

  2. Name the port Control.
    Note that the port is propagated to the remaining three forks.

  3. Align the ports on the forks so that they are facing the middle of the diagram.

  4. Save your model.

Add Interface Ports to the philosophers

You have to add left and right ports to the philosophers.  In this example you will use alternative methods to using the Palette.

To add Interface Ports to the philosopher components, follow these steps:

  1. Select a philosopher and right-click.

  2. From the context menu, select Add CX > Interface Port.

  3. Name the port leftHand.

  4. Hover over a philosopher until the Action toolbar appears.

  5. From the toolbar, select (Add Interface Port).

  6. Name this second port rightHand.
    Note that the ports are propagated to the remaining three philosophers.

  7. Align the ports on the philosophers so that the leftHand ports are on top and the rightHand on the bottom.

  8. Save your model.

Connect Ports

You now have to connect the control ports on the forks to the hand ports on the philosophers.

To connect the ports, follow these steps:

  1. From the Palette, drag-and-drop a Connection onto the fork Control port and connect it to a hand port on a philosopher.

  2. Similarly connect all the Control ports on the forks to the hand ports on the philosophers.

    Note    : You can multi-drop connections onto the diagram by holding down the CTRL key.
     

  3. Align the elements in the diagram so that it looks somewhat akin to the following:

  4. Save your model.

Define and Add the CORBA Interfaces

You now need to define the Interfaces that the ports use or provide,

  1. In the Project Explorer, right-click DiningPhilosophers.

  2. From the context menu, select Add CX > Package.

  3. Name the Package objects.

  4. Right-click objects and add another package named interfaces.

  5. Right-click interfaces and from the context menu, select Add Diagram > CX Component Diagram. This will be used to graphically display the interfaces we will create in the next step.

  6. Name the diagram interfaces_diagram.

  7. Save your model.

Add a New Interface and an Operation

To add a CORBA Interface, follow these steps:

  1. From the palette, select the CORBA drawer.

  2. Drag-and-drop an Interface onto interfaces_diagram.

  3. Name the Interface ForkControl.

  4. Right-click ForkControl and from the context menu, select Add CX > Operation.

  5. Name the Operation pickup.

  6. Save your model.

Assign the Ports to the ForkControl Interface

To assign the ports to the interface, follow these steps:

  1. From the Project Explorer, select the Philosopher component diagram, Philosopher_diagram.
    Philosopher_diagram     opens.

  2. From the Project Explorer, select the ForkControl interface.

  3. To assign the leftHand port to the ForkControl interface, drag-and-drop the ForkControl interface onto the leftHand port.

  4. In the Properties tab, check Is Conjugated to indicate that this port uses the interface.

  5. Repeat for the rightHand port.

  6. Save your model.

The leftHand and rightHand ports now use the ForkControl interface.

 

Assign the Fork port to be the Provides port

To assign the ports, follow these steps:

  1. From the Project Explorer, select Fork_diagram.
    Fork_diagram     opens.

  2. From the Project Explorer, select the ForkControl interface.

  3. To assign the Control port to the ForkControl Interface, drag-and-drop the ForkControl Interface onto the Control port.

  4. Save your model.

  5. From the Project Explorer, select DiningPhilospher_asm.
    Note that the appropriate interfaces are now identified on each port.

Define and Add the DDS Connectors

So far you have been using CORBA for messaging.   Now you will use DDS messaging to design a way for the Philosopher to publish their status.

Add DDS Connectors

To add the DDS Connectors, follow these steps:

  1. From the Project Explorer, right-click objects and from the context menu select Add CX > Package.

  2. Rename the package messages.

  3. Right-click messages and from the context menu select Add CX > Message.

  4. Rename the message PhilStatus.

  5. Save your model.

Add New Package for Connections

To add a new package for Connections, follow these steps:

  1. From the Project Explorer, right-click objects and from the context menu select Add CX > Package.

  2. Rename the package connections.

  3. Right-click connections and from the context menu select Add CX > DDS Connector Type.
    The tool opens the Create DDS Connector Instance wizard.

  4. In the Message Name field, type PhilStatus_conn.

  5. Click Finish.

  6. Save your model.

Add New Interface Port to Philosopher

To add the New Interface Port to Philosopher, follow these steps:

  1. Select Philosopher_diagram.

  2. From the Palette, drag-and-drop an Interface Port onto Philosopher.

  3. Rename the Interface Port status.

  4. In the Properties tab below the diagram, click (Edit Porttype).
    The tool opens the Element Selection Dialog.

  5. From the Matching items list, select DDS_Write-DiningPhilosophers::objects::connections::PhilStatus_conn_conn::DDS_Write.

  6. Open DiningPhilosopher_asm and note that the port has been added to all the philosopher components.

  7. Click OK.

  8. Save your model.

The Philosopher will write the status out using this connection.

Create a Data Space

To create a data space, follow these steps:

  1. In DiningPhilosopher_asm, align the DDS_Write ports so that they are on the inside of the philosopher components.

  2. From the Palette, drag-and-drop a DataSpace onto the middle of the diagram.

  3. In the Properties tab below the diagram, click (Edit Connector type).
    The tool opens the Element Selection Dialog.

  4. From the Matching items list, select DDS_Event-DiningPhilosophers::objects::connections::PhilStatus_conn_conn::DDS_Event.

  5. Click OK.

  6. Save your model.

Connect DDS_Write Ports to the Data Space)

To connect the DDS_Write Ports to the DDS4CMM Connector, follow these steps:

  1. From the Palette, drag-and-drop a Connection from a DDS_Write Port to the DataSpace.

  2. Repeat for the remaining three DDS_Write Ports.

  3. Save your model.
