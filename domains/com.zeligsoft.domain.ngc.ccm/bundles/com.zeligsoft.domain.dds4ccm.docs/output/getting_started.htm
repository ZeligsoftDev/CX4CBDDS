Tutorial 1: Getting Started

In this tutorial you will learn to:

  • Create Projects and Models

  • Import Model Libraries Required for your Design

  • Set Preferences

  • Save and Validate

You can view a video of this tutorial here.  Depending on your screen size, you may want to close the Contents frame.

Create Projects and Models

Models are saved in EMX format. A single project can have multiple models.  You can use the elements from different models within one project. Additionally, you may want to have separate platform and application models. However, for simplicity, this tutorial will use just one model.

To create a new project and model, follow these steps:

  1. Select File> New > New DDS4CCM Project.
    The tool opens the New DDS4CCM Project wizard.
  2. In the Project name field, type DiningPhilosophers and click Next.
  3. In the Model Name field, type DiningPhilosophers.
  4. Click Finish.
    The tool creates the project and model, and displays them in the Project Explorer tab.  General information is displayed in the main tab.
  5. Click DiningPhilosophers in the Project Explorer tab.
    The folder expands to reveal two folders: Diagrams and Models.
  6. Click Models.
    The folder expands to reveal the DiningPhilosophers model, including its default model configuration and IDL primitives, which are automatically generated.
  7. Save your model.

The Diagrams and Model folders provide a logical view of elements in your model.  They are not actual folders on the file system.

Import Model Libraries Required for your Design

You need to add the DDS DCPS Connector to the Model Library.  The DCPS Connector is specified in the standard; however, it is not mandatory.  

To add a DCPS Connector to the Model Library, follow these steps:

  1. Right-click on the DiningPhilosophers model and select Open.
    The tool presents general information about the model in the main tab.

  2. Click the Details tab at the bottom of the main tab.

  3. In the Model Libraries pane, click Add...
    The tool opens the Import Model Library dialog.

  4. From the pull-down list under Deployed Library, select DDS DCPS Connector.

  5. Click OK.
    The tool adds the connector to the model.

  6. Save your model.

Set Preferences

Preferences are, by definition, personal.  You can choose to set them any way you want.  Nevertheless, the following are a couple of particularly useful ones.

To set preferences, follow these steps:

  1. Select Windows > Preferences.
    The tool display the Preferences dialog.

  2. Note the options.

  3. Open Modeling > Appearance and set the preferences as you like

  4. Open Modeling > UML Diagrams > Composite Structure.

  5. Note the options.

Save and Validate

You should, of course, periodically save your work.  You should also periodically validate your work.

To validate your model, follow these steps:

  1. Right-click the model and select Validate.
    The tool returns Errors and Warnings in the Problems tab.

  2. In the Problems tab, click a Description to link to an Error or Warning.

Tutorial Summary

In this tutorial you have learned how to create a CBDDS project and model, set your preferences, and save and validate your work.
