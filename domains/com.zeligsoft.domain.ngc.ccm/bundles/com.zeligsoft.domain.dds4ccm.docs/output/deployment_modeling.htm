Tutorial 5: Deployment Modeling

This tutorial builds on and assumes you have completed Tutorial 1: Getting Started , Tutorial 2: Top-Down_Design and Tutorial 4: Target Modeling.

Deployment modeling consists of allocating application components to target (platform) elements in a pre-determined manner—engineering the system. The goal is to produce a valid Deployment Plan. Deployment plans consist of XML descriptor files that  are used configure the component framework so that it deploys an application onto a target environment. In this case, the framework is CCM and the deployment engine is DAnCE.

In this tutorial, you will

  • Create a Deployment Plan for the Application

Create a Deployment Plan for the Application

A Deployment Plan maps application components to processes on the Target Nodes. For a given Application and Target, it is desirable to be able to model multiple deployments. CBDDS provides a Deployment model element for this purpose.

To create a deployment plan for the application, follow these steps:

  1. Right-click the DiningPhilosophers model and from the context menu select Add CX > Package.

  2. Rename the package deployments.

  3. Right-click deployments and from the context menu select Add CX > Deployment Plan.
    The tool creates a deployment plan.

  4. Rename the Deployment Plan FourPhilosophersOnTwoProcessors.

  5. Open the deployment and note that the tool has created two lists: Configure and Deploy.  The Configure list comprises the elements of the model that you want to deploy.  The Deploy list comprises the assignment of application components to the target platform elements.

  6. Drag-and-drop the platform, DualProcessorSystem, and the application component, DiningPhilosphers, onto the Configure list.

  7. From the Configure list, drag-and-drop DualProcessorSystem onto the Deploy list.

  8. Save your model.

Create Processes for Philosophers

The Process element acts as the container for deploying the components.

To create processes for philosophers, follow these steps:

  1. From the Project Explorer, drag-and-drop the Process element (Platform/DualProcessorSystem/Process) onto the Configure list.  The Process element will act as the container for deploying the components.

  2. Using the Properties tab, name the process PhilosophersProcess0.  

    Note    : When creating a deployment plan naming is only done using Properties : Name.
     

  3. Create three more processes (PhilosophersProcess1, PhilosophersProcess2, PhilosophersProcess3), one for each of your philosophers.

  4. Save your model.

Assign Philosophers to Processors

To assign each philosopher to a processor, follow these steps:

  1. Drag-and-drop PhilosophersProcess0 onto a processor.

    Note    : When you hover over the processor, the list expands.

    Note: The icon for PhilosophersProcess0 in the Configure list will display a check mark to indicate that the element has been deployed.
     

  2. Repeat for the PhilosophersProcess1, PhilosophersProcess2, and PhilosophersProcess3.

  3. Drag-and-drop each Philosopher onto its associated process; for example, philosopher0 goes onto PhilosopherProcess0.

  4. Save your model.

Deploy Forks to a Processer

To create a container for the forks, follow these steps:

  1. Drag-and-drop the Process element onto the Configure list.

  2. In the Properties tab, name the process ForkProcess.

  3. Drag-and-drop ForkProcess onto a processor.

  4. Select all the fork components and drag-and-drop the four fork components onto ForkProcess.

  5. Save your model.

Note: The tool validates the assignment of components to target elements; for example, you cannot assign a fork to the bus.
