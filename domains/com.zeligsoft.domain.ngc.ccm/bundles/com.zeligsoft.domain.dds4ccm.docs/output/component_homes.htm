Component Homes

This tutorial assumes that you have completed Tutorial 1: Getting Started through Tutorial 5: Deployment Modeling.

You can view a video of this tutorial here.  Depending on your screen size, you may want to close the Contents frame.

In this tutorial you will learn how to add and deploy a Component Home.

Create a Component Home for the Fork

To add a Component Home, follow these steps:

  1. Right-click the Fork component and from the context menu select Add CX > Component Home.

  2. Notice that this adds ForkHome into the Fork package and that it Manages the Fork type.

  3. To show this relationship graphically, drag-and-drop ForkHome onto the Fork_diagram.

  4. Save your model.

Add the Component Home to the Assembly for the Forks

Component Homes need to be added to assemblies of their component types in order to manage the component instances.

To do this, follow these steps:

  1. Drag-and-drop ForkHome onto the DiningPhilopsophers_asm structure diagram. This part is not connected to the Fork parts.

    Note    : The Name will change to forkHome to adhere to the naming standard.

  2. Save your model.

Add the Component Home to the Deployment Plans

Component Homes need to be deployed in the same Container Process in order to manage the component instances.

To deploy a Home, follow these steps:

  1. Return to the FourPhilosophersOnTwoProcessors Deployment Plan. forkHome will now be displayed in the Configure list without a check mark indicating it needs to be deployed.

  2. Drag-and-drop forkHome onto the ForkProcess in the Deploy list.

  3. Right-click FourPhilosophersOnTwoProcessors and from the context menu select Generate Descriptor Files.

  4. Save your model.
