Introduction

Zeligsoft CX is a Model-based Software Engineering (MBSE) development tool that can be specialized to the exact needs of a domain. Domain specialization is essential for the design of re-usable software components, which take advantage of standardized architectures, optimized design patterns, and system-specific platform technologies.  CX applies state-of-the-art Model-based Software Engineering (MBSE) techniques and practices to the design and development of the application and platform layers of an engineered system. By modeling advanced multiprocessor, multicore and System on Chip (SoC) architectures, CX provides the design, development and maintenance support for system structure, and inter-component communication and control, allowing developers to minimize software complexity and rapidly adapt software to new platforms.

CX is built on the Eclipse and Rational Software Architect (RSA) development environments. CX has been designed to integrate fully with these development environments—taking advantage of key RSA and Eclipse concepts and capabilities. CX is an extension of this functionality, giving you the benefit of a system-centric software development tool essential for the design of re-usable software components.  CX's powerful visual system representation is UML 2.0 compliant, which gives cross-functional and geographically-dispersed team members critical insight into system structure.

CX is a domain-specific modeling environment. CX provides developers with highly-specialized development perspectives that enable them to model, configure, integrate, and validate applications and platforms through a UML-based interface that is optimized for their type of application.
