Generating IDL and Descriptors

CBDDS provides correct-by-construction generation of the XML and IDL artifacts modeled previously.

You can view a video of this tutorial here.  Depending on your screen size, you may want to close the Contents frame.

Generating IDL

At any point during the Design process you may choose to generate IDL from the model.

To generate IDL, follow these steps:

  1. Right-click any (Component) or CORBA object and from the context menu select Generate IDL.
    The tool opens the Running Transformation dialog and generates the IDL.

  2. In the Project Explorer, open the DiningPhilosophers_generatedSRC folder, where you can find all the generated IDL files.

Note: CBDDS employs iterative generation so the artifacts are always reflective of the model.

Generate the Descriptors for a Deployment Plan

Once you have completed the design of an application and created a deployment plan, you can generate the descriptor files required by the DAnCE deployment engine. The following artifacts are generated by CBDDS:

  • The Component Deployment Plan (.cdp file) that maps each component instance in the assembly’s Component Implementation Description to a logical node.

  • The Component Domain Descriptor (.cdd file) that describes available nodes, interconnects, and bridges.

After completing Tutorial 5, generate descriptors by following these steps:

  1. Right-click FourPhilosophersOnTwoProcessors and from the context menu select Generate Descriptor Files.
    The tool opens the Running Transformation dialog and generates the IDL.

  2. In the Project Explorer, open the DiningPhilosophers_generatedSRC folder, where you can find the .cdp file for the deployment and the .cdd file for the platform.
