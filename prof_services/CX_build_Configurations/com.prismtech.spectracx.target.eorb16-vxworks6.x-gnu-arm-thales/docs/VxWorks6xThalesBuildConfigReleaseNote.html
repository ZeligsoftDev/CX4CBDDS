VxWorks 6.x.0 e*ORB 1.6 ARM Build Configuration Release Notes

Version: 0.1.1

Release Date: August 2010

System Requirements

Host Operating System Linux (Red Hat Enterprise 5.4)
Target Operating System VxWorks 6.8 or 6.9
Supported BSPs OMAP 35XX

Environment Variables

The following environment variables must be set before starting Spectra CX.

  1. EORBHOME - set to the directory where e*Orb is installed.
  2. EORBTARGETENV - set to the following value: vxworks-68-gnu-arm-linux or vxworks-69-gnu-arm-linux
  3. The VxWorks environment variables need to be set
    1. WIND_BASE
    2. WIND_USR
    3. WIND_HOME
    4. WIND_GNU_PATH
    5. WIND_HOST_TYPE

Usage Instructions

Before starting Spectra CX, ensure that the WindRiver environment has been set up, and that the version of make on the PATH is version 3.81.

The following changes have to be made to components modeled in Spectra CX to generate correct XML for deployment:

  • Change the SCA code properties of each component implementation to the following:
    • Filename: ensure this ends in .vxe
  • Set the OS dependency name to VxWorks and version to 6.8.0 or 6.9.0
  • Set the processor dependency name to ARM
  • Update the os_name, os_version and processor_name properties of the Devices and DeviceManagers to the above values.

Known Issues

Upgrading from version 0.1.0

Users upgrading from version 0.1.0 of this build configuration may find that the compiler defines aren't updated correctly. This is due to Spectra CX caching the previous values. To ensure the new values are correctly picked up users should do the following for each project which uses this build configuration:

  1. Delete the existing build configuration import from your model (if still shown) and re-import the new new build configuration.
  2. On each modeled implementation switch to the new build configuration.

Monitor may timeout when creating applications

If the monitor reports a failure when creating an application, but no errors are visible in the VxWorks console, this may just be Spectra CX reaching a time limit. Try closing the monitor and re-opening after a short time to check whether the application is available.

This is caused by a bottleneck in the TCP/IP loopback mechanism which is used by the ExecutableDevice when loading resources. This, coupled with resource file sizes of around 2MB, can result in application creation times of around 25 seconds when deploying the tutorial waveform.
