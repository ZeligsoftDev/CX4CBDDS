Environment Variables

The following environment variables must be set for all build configurations before starting Spectra CX.

  1. EORBHOME - set to the directory where Spectra ORB is installed.
  2. BIOS_INSTALL_DIR - set to the directory where DSP/BIOS is installed.
  3. C6X_INSTALL_DIR - set to the directory where CGTools toolchain is installed.

In addition to use the DTP4700 build configurations the following environment variables must also be be set:

  1. XDCTOOLS_DIR - set to the directory where the XDCTools package is installed.
  2. DSPLINK - set to the directory where the DSPLink package is installed.
  3. DVSDK - set to the directory where the TI DVSDK software is installed.

In addition to use the PRO-4600 build configurations the following environment variables must also be be set:

  1. CSL_INSTALL_DIR - set to the directory where the CSL package is installed.
  2. SSP_INSTALL_ROOT - set to the directory where the Spectrum Signal's software is installed.

Usage Instructions

The following settings are required for components modeled in Spectra CX to generate correct XML for deployment:

  • The SCA code properties of each component implementation should be set to:
    • Filename: ensure this ends in .out
  • The OS dependency name should be set to DSPBIOS and version left blank
  • The processor dependency name should be set to C6416
  • The os_name and processor_name of the ExecutableDevice responsible for loading the DSP should match the above values.

Build Configuration Restrictions

  • The build configurations contained in this target support plug-in support the generation of SCA Resource and ResourceFactory components only.
  • DSP processors are limited to running a single binary image at any one time. In terms of the SCA this means that only a single component instance can be deployed to a DSP processor at a time. This limitation can be overcome by using a ResourceFactory component to co-locate all Resource components to be deployed simultaneously into a single binary image.
