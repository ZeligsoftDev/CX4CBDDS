VxWorks 5.5.1 e*ORB 1.6 PPC Build Configuration Release Notes

Version: 1.1.0

Release Date: August 2010

System Requirements

Host Operating System Windows XP (x32)
Target Operating System VxWorks 5.5.1
Supported BSPs ppc604 and ppc85xx

Environment Variables

The following environment variables must be set before starting Spectra CX.

  1. EORBHOME - set to the directory where e*Orb is installed.
  2. EORBTARGETENV - set to one of the following values based on chipset:
    • PPC604 - vxworks-551-gnu-ppc604-win32
    • PPC8540 - vxworks-551-gnu-ppc8540-win32
    • PPC8572 - vxworks-551-gnu-ppc8572-win32
  3. VXWORKS_CHIPSET - this should be set to 604 for the PPC604 board and 8540 for the PPC85XX family.

Usage Instructions

The following changes have to be made to components modeled in Spectra CX to generate correct XML for deployment:

  • Change the SCA code properties of each component implementation to the following:
    • Filename: ensure this ends in .out
    • Entrypoint: oe<component-name>. For example, oeAssemblyController.
    • Type: Shared Library
  • Set the OS dependency name to VxWorks and version to 5.5.1
  • Set the processor dependency name to PPC
  • Update the os_name, os_version and processor_name properties of the Devices and DeviceManagers to the above values.
  • The log service file name must be LogService.out and the entrypoint is LogService_main.

New features in 1.1.0

Support for Spectra CX 3.2.1

The build configuration has been updated to support Spectra CX 3.2.1.

Known Issues

Upgrading from version 1.0.0

Users upgrading from version 1.0.0 of this build configuration may find that the compiler defines aren't updated correctly. This is due to Spectra CX caching the previous values. To ensure the new values are correctly picked up users should do the following for each project which uses this build configuration:

  1. Delete the existing build configuration import from your model (if still shown) and re-import the new build configuration.
  2. On each modeled implementation switch to the new build configuration. The list of available build configurations will contain two VxWorks 5.5.1 configurations with exactly the same textual description. Choose the 2nd in the list. Users can verify they have chosen the correct configuration by ensuring the Compiler Defines section contains the define "FLAT_ADDRESS_SPACE".

CDT reports false errors when building for 85XX CPUs

When building for the 85XX family of CPUs the compiler issues a warning for function calls to deactivate_object within the generated port code. However the C++ Development tools within Spectra CX incorrectly report this compiler warning as an error. This will show up as two separate problems in the Problems view: an error with the description "warning:" and a warning with the description "choosing". Any reported errors of this kind can safely be ignored as they do not prevent the build from completing successfully.
