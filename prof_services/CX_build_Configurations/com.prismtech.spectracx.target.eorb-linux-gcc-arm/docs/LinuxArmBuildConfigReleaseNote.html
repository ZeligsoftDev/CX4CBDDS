Environment Variables

The following environment variable must be set for all build configurations before starting Spectra CX:

  1. EORBHOME - set to the directory where Spectra ORB is installed.

In addition to use the DTP4700 build configurations the following environment variables must also be be set:

  1. DVSDK - the directory where the DVSDK is installed.
  2. DSPLINK - the directory where DSPLink is installed.

Usage Instructions

Before starting Spectra CX, ensure that the PATH includes the directory containing the cross-compiler tool chain.

In order to deploy the tutorial example using the PrismTech Core Framework on the supported platforms the following changes must be made to the components:

  • Set the OS dependency version to 2.6
  • Set the processor dependency name to ARM
  • Update the os_version and processor_name properties of the Devices and DeviceManagers to the above values.

Known Issues

Segmentation faults occur when running generated components

In certain cases the components generated by Spectra CX may encounter a segmentation fault when the code is compiled with optimisation enabled. This is an issue caused by the ARM cross compiler and not an issue with the generated code. Depending on the exact cross compiler tool chain used the level of code optimisation may need to be altered to prevent segmentation faults. This can be achieved by overriding the compiler flags on the affected components.
