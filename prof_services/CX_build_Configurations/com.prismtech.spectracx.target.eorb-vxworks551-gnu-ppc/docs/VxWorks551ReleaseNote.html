Environment Variables

The following environment variables must be set for all build configurations before starting Spectra CX.

  1. EORBHOME - set to the directory where Spectra ORB is installed.
  2. CFHOME - set to the directory where Spectra CF is installed.
  3. WIND_BASE - set to the directory where VxWorks is installed.

In addition the following environment variables must be set for the eorb17-vxworks551-gnu-ppc-pro4600-build build configuration.

  1. SSP_INSTALL_ROOT - set to the directory where the Spectrum Signal's software is installed.

Usage Instructions

The following settings are required for components modeled in Spectra CX to generate correct XML for deployment:

  • The SCA code properties of each component implementation should be set to:
    • Filename: ensure this ends in .out
    • Entrypoint: scx<component-name>_main for generated components (e.g. scxAssemblyController_main).
    • Type: Shared Library
  • The OS dependency name should be set to VxWorks and version to 5.5.1
  • The processor dependency name should be set to PPC
  • The os_name, os_version and processor_name properties of DeviceManagers, LoadableDevices and ExecutableDevices should match the above values.

VxWorks 5.5.1 Shared Libraries

A VxWorks 5.5.1 system has a single address space that is shared by all running programs. When a shared library (.out file) is loaded into the system the symbols from the library are loaded into this single address space. It is important that duplicate symbols are not loaded into the system as this can result in unexpected behavior when running programs. In order to reduce the risk of loading duplicate symbols into the system it is highly recommended that any common code shared between different programs is combined into a single shared library and loaded into the system only once.

When compiling generated code Spectra CX produces a shared library for each component containing the symbols unique to that component. In addition Spectra CX produces several static libraries (.a files) containing the symbols which are shared between different components (e.g IDL, port and framework code). Spectra CX also produces a shared library (named <Project Name>-libs.out) containing the shared ORB, CF and Spectra CX generated static libraries which are required to run the components contained within a particular generated source project. This shared library should be loaded onto the board before running generated platforms or waveforms and can be found under the obj/<tools>/<build>/cxdef folder.

To ensure correct operation of a deployed SCA system all components to be deployed simultaneously should be modeled in the same Spectra CX model to ensure the resultant <Project Name>-libs.out library contains everything required. Under no circumstances should multiple instances of these libraries be loaded into the system at the same time.

Where the common library generated by Spectra CX is not suitable users should create a single shared library for their system containing the symbols from the required ORB, CF and CX static libraries. Spectra CX generated components have the following static library dependencies:

  • ORB Libraries
    • ec_os
    • ec_orbs
    • ec_util
    • ec_orb
    • ec_tcp
    • ec_iiop
    • ec_poa
    • ec_any
    • ec_lwlog_c
    • ec_lwlog_s
    • ec_lwevent_c
    • ec_lwevent_s
    • ec_naming_c
    • ec_naming_s
    • e_orb
    • e_mpoa
    • e_any
    • e_iiop
    • e_lwnaming_c
    • e_lwevent_s
    • e_lwlog_s
  • Additional ORB Transport Libraries (Required only by the eorb17-vxworks551-gnu-ppc-pro4600-build build configuration)
    • ec_qc
    • ec_qciop
    • e_qciop
  • Core Framework Libraries
    • cf_os
    • cf_impl
    • cpporb_launcher
  • Spectra CX Libraries
    • CF
    • PortTypes
    • framework
    • executabledeviceframework (Required only for components derived from the CF::Device interface)
  • Spectra CX TargetLoader Libraries (Required for running the generated unit tests)
    • TargetLoader
    • targetloaderframework

The Spectra CX Libraries can be found under the obj/<tools>/<build>/cxdef directory. In addition to the listed libraries application specific IDL and port libraries also need to be added to the single shared library containing common code.
