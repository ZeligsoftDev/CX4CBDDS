VxWorks 6.8 e*ORB 1.6 GNU PPC DKM Build Configuration Release Notes

Version: 3.5.2

Release Date: September 2013

System Requirements

Host Operating System Linux (Red Hat Enterprise 5.4) or Windows XP
Target Operating System VxWorks 6.8
Supported BSPs SBC8548
MPC8572

Environment Variables

The following environment variables must be set before starting Spectra CX.

  1. EORBHOME - set to the directory where e*Orb is installed.
  2. EORBTARGETENV - set to the following value: vxworks-68-gnu-ppc-linux (Linux) or vxworks-68-gnu-ppc-win32 (Windows XP)
  3. CX_TARGET - set to either SBC8548 or MPC8572.

Usage Instructions

The following changes have to be made to components modeled in Spectra CX to generate correct XML for deployment:

  • Change the SCA code properties of each component implementation to the following:
    • Filename: ensure this ends in .out
    • Entrypoint: scx<component-name>. For example, scxAssemblyController.
    • Type: Shared Library
  • Set the OS dependency name to VxWorks and version to 6.8
  • Set the processor dependency name to PPC
  • Update the os_name, os_version and processor_name properties of the Devices and DeviceManagers to the above values.
  • The log service file name must be LogService.out and the entrypoint is LogService_main.

To use Spectra CX generated components the following libraries must be loaded onto the target board in order:

  • ORB Libraries
    • ec_os.out
    • ec_orbs.out
    • ec_util.out
    • ec_orb.out
    • ec_tcp.out
    • ec_iiop.out
    • ec_poa.out
    • ec_any.out
    • ec_lwlog_c.out
    • ec_lwlog_s.out
    • ec_lwevent_c.out
    • ec_lwevent_s.out
    • ec_naming_c.out
    • ec_naming_s.out
    • e_orb.out
    • e_mpoa.out
    • e_any.out
    • e_iiop.out
    • e_lwnaming_c.out
    • e_lwevent_s.out
    • e_lwlog_s.out
  • Core Framework Libraries
    • cf_os.out
    • cf_impl.out
    • cpporb_launcher.out

Before running the generated CX components the common .out libraries (found in various locations under obj/eorb16-vxworks68-gnu-ppc-dkm-tools/eorb16-vxworks68-gnu-ppc-dkm-build/cxdef) also need to be loaded onto the board. Note that IDL libraries must be loaded before any port libraries which use the IDL. Also note that the following libraries must be loaded first in this order:

  • CF.out
  • PortTypes.out
  • framework.out
  • executabledeviceframework.out (only for devices)

Please note that it is important that duplicate symbols are not loaded onto the target board. Loading duplicate symbols onto the target board can result in errors when running the components. It is highly recommended that the above common libraries are merged into a single common .out file in order to resuce the risk of loading duplicate symbols. A Makefile is provided with the Core Framework in the lib directory which can be used to create a combined common library.
